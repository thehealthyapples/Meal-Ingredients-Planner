Task:
Create a lightweight "Quick Meal" flow to help users quickly assemble a meal without creating a full recipe.

Example use case:
A user wants to quickly plan a roast dinner by entering items such as:
- chicken
- carrots
- mixed veg
- roast potatoes
- cauliflower cheese
- gravy
- mint sauce
- Yorkshire puddings
- braised cabbage

Required behaviour:
- Add a new "Quick Meal" option.
- User can enter meal components line-by-line.
- THA builds a combined ingredient list from those components.
- The result can be opened in the basket as a shopping list.

Implementation requirements:
- This must be a lightweight tool, not a full recipe creation system.
- Reuse ingredient search and basket logic already in the app.
- Do not introduce a new recipe model unless absolutely necessary.
- Treat Quick Meal primarily as a fast list builder.

UI requirements:
- Simple interface:
  - meal name optional
  - add components one at a time
  - button: "Create Basket List"

Optional if cheap to support:
- show quick wholefood suggestions for items if available.

Validation:
- Only ensure that components correctly generate a basket list.
- Do not run broad tests.

- which files changed
