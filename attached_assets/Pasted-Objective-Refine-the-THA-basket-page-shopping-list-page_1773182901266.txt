Objective

Refine the THA basket page (shopping-list-page.tsx) into the new category-based layout spec:

Replace the single flat table with category-grouped tables, each with a sticky single-line header
Category headers show: Category Name | Default Supermarket | Default Tier (or "Mixed" if any row has explicit override ≠ category default)
Rows inherit category default supermarket/tier unless overridden per-row
Column spec: Ingredient | Variant | Tier | Qty | Match | Price | Confidence | THA Rating
After all category tables: Basket Totals section (Total Price, Avg THA Rating, Overall Confidence)
Below that: Comparison Strip table (Shop × Budget/Standard/Premium/Organic/Current)
"Difference vs Current" = min(tier column across retailers) − Current Total
"Current Total" = sum of Current column (actual spend per retailer)
Pantry section: render using same category layout but NO new pantry logic
No backend changes; no pantry merge/recommend logic

Details:
Add below Basket Totals, inside or directly after the Card
Only render if hasPrices && selectedRetailers.length > 0

comparisonMatrix useMemo:
for each retailer in selectedRetailers × each tier in ['budget','standard','premium','organic']:
total = sum over savedItems of:
allPriceMatches.find(m => m.shoppingListItemId === item.id && m.supermarket === retailer && m.tier === tier)?.price ?? 0

currentByRetailer useMemo:
for each retailer → sum the price of each item attributed to that retailer using this priority:

Explicit assignment first:
if item.selectedStore === retailer, use pricesByItem.get(item.id)?.get(retailer)?.price

Auto fallback only:
if item.selectedStore is null, attribute the item to the retailer currently chosen by existing auto-match logic for that row.
Do not attribute by theoretical cheapest price unless that is the row’s actual current auto-selected retailer.

Items with an explicit selectedStore set to a different retailer are NOT counted here, even if a price exists
