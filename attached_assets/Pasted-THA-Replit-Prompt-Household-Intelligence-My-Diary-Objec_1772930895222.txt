Phase 3A — Household Basket Intelligence
Goal

Make shared household basket behaviour understandable.

Users should see:

• who added an item
• why the item exists
• which meal caused it
• household context

Schema Updates

In shared/schema.ts

Add to shopping_list table:

addedByUserId: integer("added_by_user_id")

Add snapshot planner context fields to ingredient_sources:

weekNumber: integer("week_number")
dayOfWeek: integer("day_of_week")
mealSlot: text("meal_slot")

These are snapshot metadata only and must not create tight coupling to planner edits later.

Run:

npm run db:push --force

Phase 3C — Household Dietary Context

Add endpoint:

GET /api/household/dietary-context

Aggregate across all members' user_preferences.

Return:

{
  members: [
    {
      displayName,
      dietTypes,
      dietRestrictions,
      excludedIngredients
    }
  ],
  aggregated: {
      unionDietTypes,
      unionRestrictions,
      unionExclusions
  }
}

This is read-only groundwork for later meal intelligence.

Phase 4 — My Diary (Personal Food Diary)
Naming

Sidebar label:

My Diary

Aligns with:

My Meals

My Diary belongs to userId only.

Never query by householdId.

Schema

Add tables in shared/schema.ts.

food_diary_days
id serial PK
userId integer FK users
date text (YYYY-MM-DD)
notes text
createdAt timestamptz
updatedAt timestamptz
unique(userId, date)
food_diary_entries
id serial PK
dayId integer FK food_diary_days
userId integer FK users
mealSlot text (breakfast | lunch | dinner | snack)
name text
notes text
sourceType text default 'manual'
sourcePlannerEntryId integer nullable
createdAt timestamptz

sourceType:

manual
copied_from_planner
food_diary_metrics
id serial PK
userId integer FK users
date text
weightKg real
bmi real (optional fallback)
moodApples integer (1–5)
sleepHours real
energyApples integer (1–5)
notes text
stuckToPlan boolean
createdAt timestamptz
unique(userId, date)
BMI Rule

BMI should normally be derived:

BMI = weightKg / height^2

Use user_preferences.height where available.

Persist bmi only if:

height missing
Apple Scoring System

Mood and Energy use THA Apple scoring.

1 🍏
2 🍏🍏
3 🍏🍏🍏
4 🍏🍏🍏🍏
5 🍏🍏🍏🍏🍏

These correspond to:

moodApples
energyApples

Validate range 1–5.

Diary API Routes

Add routes.

Fetch Day
GET /api/food-diary/:date

Return:

{
 day,
 entries,
 metrics
}
Copy From Planner
POST /api/food-diary/:date/copy-from-planner

Behaviour:

determine weekday from date

load planner meals for that weekday

create diary entries

Must be idempotent.

Skip entries where:

sourcePlannerEntryId already exists for that user/date
Entry Management
POST /api/food-diary/:date/entries
PATCH /api/food-diary/entries/:entryId
DELETE /api/food-diary/entries/:entryId
Metrics
PATCH /api/food-diary/:date/metrics
GET /api/food-diary/metrics/trends

Return last 90 days ordered by date.
