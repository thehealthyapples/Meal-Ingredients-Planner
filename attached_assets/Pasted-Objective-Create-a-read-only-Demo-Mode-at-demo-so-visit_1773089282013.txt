Objective
Create a read-only Demo Mode at /demo so visitors can explore THA without creating an account.

Demo mode must:
- use client-side static fixture data only
- require no backend or database changes
- be visually consistent with the main app shell
- allow browsing planner, basket, and meals pages
- block all write actions with a read-only modal
- include a sticky demo banner with Reset Demo and Create Account actions

Architecture
- Build demo as standalone lightweight pages under /demo/*.
- Do NOT wrap or reuse the existing complex planner/basket/meals pages unless a small presentational component can be safely reused.
- Demo data must live in client fixtures, not the DB.
- Add a DemoContext that exposes:
  - isDemoMode: boolean
  - resetKey: number
  - reset: () => void
- Add a useDemoMode() hook for nested components.
- Add a DemoRoute wrapper similar to ProtectedRoute that:
  - renders the normal shell
  - does not require authentication
  - inserts a sticky demo banner above the TopBar
- Reset Demo should increment resetKey so demo pages remount fresh.
