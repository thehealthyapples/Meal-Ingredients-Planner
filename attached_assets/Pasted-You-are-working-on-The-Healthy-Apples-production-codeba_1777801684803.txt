NEW BRANCH OF WORK: UNIFIED BASKET + QUICK LIST SHOP VIEW.

DO NOT CHANGE CODE.

Product decision we are exploring:
Basket should become the single source of truth for shopping.
Quick List should feed into Basket, not maintain a separate hidden Shop View.

Target mental model:
Basket = everything I need to buy
Quick List = fast way to add items
Shop View = shopping mode for Basket

Desired behaviour:
1. There should be ONE Shop View.
2. Basket Shop View should show exactly what is in Basket.
3. Quick List items should be added into Basket.
4. Basket items must retain source:
   - planned / basket / recipe / planner
   - quick_list
5. Shop View should support filters:
   - All
   - Planned
   - Quick List
6. Opening Shop View from Basket should default to All.
7. Opening Shop View from Quick List should default to Quick List.
8. User should never lose access to Quick List items from Basket.
9. User should never need to remember which Shop View they entered from.
