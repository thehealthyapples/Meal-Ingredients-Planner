#9B - Correction Pass — Visible Result UX, Sorting, Standard Modal, and No /100 Score

What & Why
This is a correction pass on the actual visible product result UI.

Previous work is not complete from a user-facing perspective. The implementation must now focus on the exact live surface the user interacts with and deliver the missing visible behaviour with no ambiguity.

Important
Every missed requirement costs money. Do not interpret this loosely.
Do not report this complete unless the visible UI on the real result surface clearly reflects every item below.

Done looks like

Visible result UI
- users can sort/group results by shop
- users can sort results by apple score
- product cards/results no longer emphasise or show a score out of 100 as a user-facing metric
- apple score is the primary visible score

Modal behaviour
- clicking a product opens a standard centered modal window
- it does NOT open as a right-hand slide-over / drawer / sheet
- the modal uses the same normal modal background/treatment as other THA modals
- the modal can be closed cleanly in the normal way

Comparison / result quality
- visible product comparison and result quality work on the actual UI surface the user is using
- if granular additive filters are part of this result surface, they must be visible and usable there
- wrong-language products must not appear in that same surface

Critical instruction
You must inspect and modify the exact live component(s) that render the product results and product open/click interaction I am using.
Do not make hidden/helper/backend-only changes and call the task complete.

Out of scope
- homepage/auth/demo/onboarding work
- broad redesign of unrelated pages
- heavy animation
- backend redesign beyond what is necessary to support the visible UI correctly

Tasks
