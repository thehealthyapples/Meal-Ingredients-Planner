Part 1 — Row Actions Auto-Reveal

Keep the Actions column present at all times so the table grid does not shift.

However, the controls inside the Actions cell should be visually hidden by default on desktop and revealed when the user interacts with the row.

Required behaviour

On desktop:

default row state: actions hidden

row hover: actions reveal

selected row: actions remain visible even when not hovered

focused row / focused control: actions remain visible

The Actions column should still occupy its full width at all times.
Only the controls inside it should fade/show.

Actions included

The Actions cell currently contains:

[selector] [analyse] [edit] [remove]

Keep this structure unchanged.

Implementation direction

Use row-level group hover/focus state rather than mounting/unmounting controls.

Example direction:

<tr className="group">

and inside Actions:

<div className="row-actions opacity-0 group-hover:opacity-100 group-focus-within:opacity-100 transition-opacity">

Also ensure selected rows force visibility.

Important requirements

no layout shift

no column width shift

no change to selection logic

no change to analyse / edit / remove behaviour

controls must remain keyboard accessible

if a child control receives focus, the actions must stay visible

Mobile / touch behaviour

Hover does not exist on touch devices.

For mobile / touch, keep Actions visible by default unless there is already an existing low-cost reveal pattern.
Do not create a large mobile interaction redesign.

Preferred low-cost behaviour:

desktop: reveal on hover/focus/selected

mobile: always visible

Part 2 — Qty Stepper Auto-Reveal

Make the Qty cell calmer by hiding stepper controls until interaction.

Required behaviour

On desktop:

default row state: show compact qty value only

row hover: reveal minus / plus controls within the Qty cell

qty cell focus / focus within: reveal controls

selected row: optional to reveal controls if easy, but not required

if the qty is actively being edited, controls remain visible

On mobile / touch:

keep qty controls visible by default if hover is not available

do not invent a complex tap-to-reveal interaction

Desired visual pattern

Default:

2

Hover/focus:

[-] 2 [+]
Important requirements

keep the Qty column width fixed

do not allow layout shift when controls appear

keep quantity centred and readable

use opacity / visibility transitions rather than changing table structure

preserve current quantity logic and handlers

preserve direct edit/input behaviour if it already exists

Implementation direction

Use the same row hover / focus pattern as Actions where practical.

Example approach:

keep a qty wrapper with reserved space

keep value always rendered

reveal stepper buttons with opacity transition

avoid changing the cell width or table layout

If needed, use an inner flex wrapper such as:

<div className="flex items-center justify-center gap-1">

with controls faded in on hover/focus.
