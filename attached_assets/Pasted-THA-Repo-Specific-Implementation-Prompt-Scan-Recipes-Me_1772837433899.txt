T303 — Recipe / Meal-plan parser service

Create:

server/services/recipeParser.ts

Export:

type ScanResult =
  | {
      type: 'recipe';
      title: string;
      servings: number;
      ingredients: string[];
      steps: string[];
      confidence: 'high' | 'low';
    }
  | {
      type: 'meal_plan';
      days: { day: string; meals: string[] }[];
      confidence: 'high' | 'low';
    }
  | {
      type: 'unknown';
      rawText: string;
    };

parseScannedText(text: string): Promise<ScanResult>
Base parser: regex / heuristics

This must always work without API keys.

Recipe detection

Treat as recipe if:

no clear day-name structure is found

text looks like title + ingredient/method content

Heuristics:

first non-empty line = title

detect headings like:

ingredients

method

steps

directions

bullet / dash / numbered lines become ingredients or steps depending on section

if no headings exist, infer:

short ingredient-like lines before longer imperative lines = ingredients

numbered lines / imperative lines = steps

Servings:

detect from text like:

serves 4

servings: 2

makes 6

fallback to 1

Confidence:

high if title + at least 2 ingredients or steps found

otherwise low

Meal plan detection

Treat as meal plan if text contains day names like:

Mon / Monday

Tue / Tuesday

Wed / Wednesday

Thu / Thursday

Fri / Friday

Sat / Saturday

Sun / Sunday

Heuristics:

split by line

detect line starting with day name

extract meal text after colon, dash, or whitespace

if a day line contains multiple items, split on commas or |

Example:

Mon: Pasta
Tue: Soup, Salad
Wed Chicken Curry

Should become:

{
  type: 'meal_plan',
  days: [
    { day: 'Monday', meals: ['Pasta'] },
    { day: 'Tuesday', meals: ['Soup', 'Salad'] },
    { day: 'Wednesday', meals: ['Chicken Curry'] }
  ],
  confidence: 'high'
}
OpenAI enhancement

If process.env.OPENAI_API_KEY exists:

create an OpenAI client

use gpt-4o-mini

send OCR text and request strict JSON matching ScanResult

validate returned JSON before using it

if OpenAI fails for any reason, fall back to regex parser automatically

Do not make OpenAI a hard dependency for successful scanning.

Acceptance:

OpenAI path enhances accuracy if available
