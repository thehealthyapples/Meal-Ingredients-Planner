Objective

Refine the THA Planner UX to use a cleaner matrix layout and centered meal modal, while preserving all existing planner logic, routes, data flows, and actions. This is a presentation / interaction refactor only.

Core UX rules

1. The planner remains a matrix:
   - full day names across the top: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday
   - meal categories down the left: Breakfast, Lunch, Dinner, Snacks, Drinks, Kids, Baby

2. Each planner cell should contain only:
   - meal name(s)
   - a small "+" control

3. Do NOT show inside planner cells:
   - THA score
   - calories
   - add to basket buttons
   - replace/edit buttons
   - extra action rows
   - basket / modal / control buttons at the bottom of each cell

4. Meal details should appear only when the user clicks a meal, via a centered modal dialog.

5. The bottom panel on the planner page should show only daily summary for the selected day:
   - total calories for that day
   - overall THA score for that day
   It must NOT show individual meal details.

6. No functional logic changes.
