#7 - Homepage Hero Page — Orchard Background, Final Copy, and Entry Flow

What & Why
Create a proper homepage hero page for www.thehealthyapples.com so the site has a true front door before auth. The homepage should carry the brand story, product purpose, and main entry actions, while the auth page remains focused on account creation/login.

The homepage hero should use the same orchard background for brand continuity and present the final agreed THA messaging and CTA structure.

Done looks like

Homepage role
- The homepage acts as the main marketing/front-door page for The Healthy Apples
- It is distinct from the auth page
- It introduces the product clearly and calmly
- It leads users to either create an account or explore the demo

Hero background
- Use the same orchard background already used in the brand/auth experience
- Keep the background soft and readable behind text
- Reuse existing background assets/patterns if already present
- Add the subtle THA apple watermark behind the text area if it supports the design cleanly

Hero content
The hero includes this exact content structure:

Top brand line:
- "Confidently better choices, simply"

Main headline:
- "Plan meals. Analyse ingredients. Build smarter baskets."

Supporting copy:
- "The Healthy Apples helps you make better food choices with less effort by combining meal planning, ingredient analysis, and practical alternatives for everyday life."

Primary CTA row:
- "Create account"
- "Explore The Healthy Apples →"

Micro-copy below CTA row:
- "Instant access • No signup • 20-minute demo"

Human story line:
- "Born from our family’s journey through health challenges and years of learning how food choices affect everyday life."

Hero visual
- Include the refined THA lifecycle visual alongside the hero copy
- This should not be a one-way arrow
- It should represent the circular lifecycle:
  - Kitchen
  - Plan
  - Analyse
  - Basket
  - back to Kitchen
- Keep the visual composed, premium, and UI-like rather than a generic infographic
- Maintain the desirable pantry/kitchen cues already developed

Layout
Desktop
- Two-column hero layout
- Left: copy and CTAs
- Right: lifecycle visual

Mobile
- Single-column stack
- Copy first
- Visual beneath

Buttons
- Primary button: "Create account"
- Secondary button: "Explore The Healthy Apples →"
- Secondary CTA should feel premium and inviting
- Keep the subtle hover treatment already discussed for the explore CTA:
  - tiny lift
  - slightly stronger shadow
  - arrow nudges right slightly

Page structure below hero
Do not overbuild the homepage. Add only light supporting sections if needed for a complete front page.

Recommended minimal sections below hero:
1. Three-value block section:
- Plan meals
- Analyse ingredients
- Build smarter baskets

2. Short purpose/story section:
Use the shorter About-style copy already agreed in tone, not the full long biography

3. Final CTA section:
- Create account
- Explore The Healthy Apples →

Scope safety
- Keep this pass focused on creating the homepage and wiring it into the site cleanly
- Do not redesign the app itself
- Do not redesign auth in this pass beyond linking to it correctly
- Do not introduce heavy new libraries
- Do not create a bloated marketing site

Out of scope
- Broad redesign of auth page
- Demo implementation changes
- Onboarding changes
- Full CMS/blog/about architecture
- Heavy animation
- Backend changes unless a tiny route wiring change is required
